# Idea to Spec

1. Read `docs/idea.md` fully.
2. If empty or placeholder-only, ask the user to fill it first.
3. Produce `docs/spec.md` with: problem statement, goals, non-goals, scope, architecture, user flows, success criteria, tech decisions.
4. Write the result to `docs/spec.md`.
5. Do not start implementation until the user approves the spec.
